GIT常用命令大全

发表于 2019-08-02

总结工作中常用的GIT命令

配置

git config --global user.name [name] - 设置全局用户名
git config --global user.email [email] - 设置全局邮箱
git config --local user.name [name] - 设置当前项目用户名
git config --local user.email [email] - 设置当前项目邮箱
git config --global list - 查看全局配置
git config --local list - 查看当前项目配置
ssh-keygen -t rsa -C "email" - 生成ssh密钥

新建仓库

git init - 在当前目录初始化一个git仓库
git clone [url] - 在当前目录克隆一个项目

添加/删除文件

git add [file] [dir] ... - 添加指定文件或目录到暂存区
git add . - 添加当前目录所有文件到暂存区
git add -u [file] - 将修改的、删除的文件添加到暂存区，排除新增的文件
git rm [file] - 删除文件并将删除放进暂存区
git rm --cached [file] - 停止追踪文件，不删除
git mv [file-original] [file-renamed] - 文件重命名

代码提交

git commit [file1] [file2] ... -m [message] - 提交指定文件到仓库
git commit -m [message] - 提交所有暂存区文件到仓库
git commit -am [message] - 直接提交工作区文件到仓库
git commit -v - 提交时显示所有diff信息

分支

git branch - 列出本地所有分支
git branch -r - 列出远程所有分支
git branch -a - 列出本地和远程所有分支
git branch [branch] - 新建一个分支
git checkout -b [branch] - 新建一个分支并切换到该分支
git branch [branch] [branch] - 在指定commit上新建一个分支
git checkout [branch] - 切换到指定分支
git branch -d [branch] - 删除分支
git push origin -d [branch] - 删除远程分支

合并代码

git merge [branch] - 合并指定分支到当前分支，有merge记录
git cherry-pick [commit] - 合并指定commit到当前分支
git rebase -i HEAD~2 - 将本地分支未提交的多个commit合并为一个
git rebase -i - 将本地分支未提交所有commit合并为一个
git rebase [branch] - 合并指定分支到当前分支，没有merge记录

标签

git tag - 列出所有tag
git tag [tag] - 在当前分支新建一个tag
git tag [tag] [commit] - 在指定commit上新建一个tag
git tag -d [tag] - 删除本地tag
git push origin :refs/tags/[tag] - 删除远程tag
git push [remote] [tag] - 提交指定tag
git push [remote] --tags - 提交所有tag
git checkout -b [branch] [tag] - 新建一个branch指向某个tag

查看变更

git status - 查看工作区变更的文件
git log - 查看当前分支commit历史
git log --graph - 图形化查看commit历史
git log -n4 - 查看最近4条记录
git log --all - 查看所有分支commit历史
git blame [file] - 查看指定文件修改历史

远程同步

git fetch [remote] - 同步远程仓库所有改动
git remote -v - 显示关联的所有远程仓库
git pull [remote] [branch] - 拉取远程仓库变动与本地分支合并
git remote add [remote] [url] - 新增一个远程仓库
git push [remote] [branch] - 推送本地变动到远程仓库
git push [remote] [branch] --force - 强行推送本地变动到远程仓库

撤销改动

git checkout [file] - 撤销指定文件的改动
git checkout . - 撤销所有文件的改动
git reset HEAD [file] - 从暂存区恢复指定文件到工作区
git reset HEAD . - 从暂存区恢复所有文件到工作区
git reset --hard [commit] - 重置当前分支为指定commit
git stash - 保存当前工作区改动至堆栈
git stash save [tips] - 保存当前工作区改动至堆栈并添加注释
git stash list - 查看当前stash中的内容
git stash pop - 将最近保存的一条stash取出至当前分支并删除堆栈中内容
git stash apply - 将最近保存的一条stash取出至当前分支保留堆栈中内容
git stash apply [stash] - 取出堆栈中指定的记录到当前分支
